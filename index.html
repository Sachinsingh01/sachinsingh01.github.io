Introduction

The purpose of this milestone report is to show the progress of the Capstone project. The Capstone project is part of Data Science Specialization offered by Coursera in close collaboration with SwiftKey, It uses HC Corpora dataset. The objective of this project is to build a predictive text model by using HC corpora database to predict the next word (on the basis of the previous words used earlier) to the user during writing. This report demonstrates the preliminary exploratory analysis of the data and the suggestions to build the prediction algorithm.

The SwiftKey database has below three data files. This data comes in German, Finnish and English languages but for this project we are going to use English language dataset only .

  1. en_US.blogs.txt - Contains Blog Posts
  2. en_US.news.txt - Contains News articles.
  3. en_US.twitter.txt - Contains Twitter tweets.

The Coursera-SwiftKey.zip file is unzipped to extract the English database as a corpus. This consists of three text files of twitter, blog and news. The data files are too big so in order to get faster data processing data samples from all three source files are generated.

Tokenization

The purpose of the tokenization is to suppress low frequency words and the meaningless characters in the corpus. Basically Tokenization is the way to break down the sentences into words and explains how sentences are formed. In Natural Language Processing (NLP) an n-gram is a contiguous sequence of n items from a given sequence of text or speech. The final corpus will show the words or n-gram with a high frequency which will be helpful for exploring the relationship between the words and building a statistical model. In this section ASCII chars are extracted, punctuation are removed, converted chars to lowercase etc.
